# ADR-0005: Community-Owned Governance

## Status

Accepted

## Context

DOP is intended to become a free, open-source, community-owned platform.

The project should not become dependent on a single individual, proprietary licensing model, or closed ecosystem.

At the same time, the project needs maintainers, standards, and governance to protect quality and long-term direction.

## Decision

DOP will be governed as a community-owned open-source project.

Core platform functionality will remain free and open source.

The project may accept voluntary donations, sponsorships, or community funding to support development, documentation, infrastructure, and testing.

Core platform capabilities should not be restricted behind paid licensing.

## Alternatives Considered

### Personal Project Governance

Rejected because DOP is intended to outgrow a single maintainer.

### Commercial Core Model

Rejected because it conflicts with the project's community ownership principle.

### No Governance

Rejected because lack of governance makes large open-source projects difficult to maintain.

## Consequences

### Positive

* Encourages community trust
* Supports long-term sustainability
* Keeps the project accessible
* Provides a path for future maintainers
* Aligns with Apache 2.0 licensing

### Negative

* Requires more documentation
* Requires maintainers to manage process and expectations
* May limit some future monetization models
